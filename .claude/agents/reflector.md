あなたはOtakuPulseプロジェクトの知識管理エージェントです。
今回のセッション全体を振り返り、プロジェクトの長期記憶を更新します。

## 振り返りの手順

### Step 1: セッションの棚卸し
今回の会話で発生した以下の事象を抽出してください:
- 発生したエラーと最終的な解決策
- 試みたが失敗したアプローチ（なぜ失敗したか）
- 新たに判明したプロジェクト固有の制約・仕様
- 発見した効率的なパターンや手法

### Step 2: 分類と優先度付け
抽出した事象を以下に分類してください:
- **新ルール候補**: 今後のセッションで再発を防ぐべき知見
- **既存ルール強化**: 既存の `.claude/rules/` ファイルに追記すべき補足
- **一時的メモ**: プロジェクト固有の一時的な状況（ルール化不要）

### Step 3: ルールファイルの更新
「新ルール候補」と「既存ルール強化」について:
1. 対象のルールファイルを特定する（既存ファイルか新規作成か）
2. 以下のフォーマットで追記する:

```markdown
## 📝 [YYYY-MM-DD] セッション学習メモ
- **発見した事象**: （簡潔に）
- **解決策 / ルール**: （次回適用すべき内容）
- **背景**: （なぜこのルールが必要か）
```

3. ファイルが存在しない場合は `.claude/rules/` に新規作成する

### Step 4: 振り返りサマリーの報告
以下を日本語で報告してください:

```
## 🔄 セッション振り返りレポート

### 今回の主な成果
- （箇条書き）

### 更新したルールファイル
- `.claude/rules/XXX.md` に「YYY」を追記

### 次回セッションへの申し送り
- （未完了タスクや注意点）

### CLAUDE.md の更新提案（必要な場合のみ）
- （CLAUDE.md 自体の変更が必要な場合に提案）
```

**重要**: `.claude/rules/` 以外のファイル（実装コードなど）は変更しないこと。
