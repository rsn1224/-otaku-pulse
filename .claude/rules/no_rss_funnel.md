# rss-funnel は採用しない

## 却下理由

| 観点 | 問題 |
|------|------|
| ライセンス | **GPL-3.0** — アプリ全体が GPL 汚染される。MIT/Apache-2.0 で配布できなくなる |
| メンテナンス | 最終リリース 2024年8月 — 1年半以上メンテされていない |
| 対応範囲 | RSS 専用 — AniList GraphQL API や Steam API には非対応 |
| 設定方式 | YAML 設定ファイル — 動的なフィード管理（ユーザーが UI から追加・削除）との相性が悪い |
| アーキテクチャ | プロセス分離が前提 — Tauri アプリへの組み込みには不自然 |

## 代替方針

以下の MIT ライセンスクレートで自前実装する:

- **feed-rs** (MIT) — RSS/Atom フィードのパース
- **scraper** (MIT) — HTML スクレイピング（OGP 取得等）

### メリット

- ライセンスがクリーン（MIT）
- フィード以外のソース（AniList, Steam）も統一的に扱える
- Tauri プロセス内で直接実行できる
- 動的なフィード設定（DB 管理）と自然に統合できる
