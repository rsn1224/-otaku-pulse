## Spacing Scale

Source: design.md §4. 8px grid, all values on 4px sub-grid.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Icon-to-text gaps, minimum inline padding |
| sm | 8px (0.5rem) | Tab gaps, icon spacing, badge padding |
| md | 12px (0.75rem) | Component internal padding (3x sub-unit exception) |
| lg | 16px (1rem) | Card gaps, section spacing |
| xl | 24px (1.5rem) | Card padding, feed column padding |
| 2xl | 32px (2rem) | Featured card padding |
| 3xl | 48px (3rem) | Section-level gaps |

Exceptions:
- `md` is 12px (not 16px) — retained from v1.0 as a 3x base half-step. New code uses this value for component-internal padding only. [Source: design.md §4]
- Touch target minimum: 44px for icon-only buttons (TopBar height = 44px = layout constant). [Source: design.md §4 Layout Constants]

## Typography

Source: design.md §3. Noto Sans JP Variable, 3-weight system. 4 type roles.

| Role | Size | Weight Token | Weight | Line Height | Usage |
|------|------|-------------|--------|-------------|-------|
| Meta / label | 11px (0.6875rem) | `--font-weight-light` | 300 | 1.4 | Timestamps, category labels, badge text |
| Body / UI text | 13px (0.8125rem) | `--font-weight-regular` | 400 | 1.5 | AI summaries, descriptions, input values |
| Card title | 15px (0.9375rem) | `--font-weight-semibold` | 600 | 1.3 | Article titles, card headings |
| Heading / modal | 18px (1.125rem) | `--font-weight-semibold` | 600 | 1.2 | Modal titles, panel headings, section headers |

Declared weights: **300 (light)** and **400 (regular)** and **600 (semibold)** — 3 weights total.
Rule: Body uses weight 400. Headings and active states use weight 600. Meta/labels use weight 300.

Font family declaration: `font-family: var(--font-jp)` — never hardcode `"Noto Sans JP Variable"` inline.

## Color

Source: design.md §2 + globals.css `:root`. All values confirmed present in globals.css.

### 60-30-10 Distribution

| Role | Token | HEX | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--surface-base` + `--surface` | `#0a0a0f` / `#14141a` | Page background, absolute void base |
| Secondary (30%) | `--surface-container` through `--surface-elevated` | `#1c1c24` → `#3b3b4a` | Cards, panels, sidebar, nav, modals |
| Accent (10%) | `--primary` | `#bd93f9` | CTA buttons, active nav state, AI badge chip |
| Destructive | `--error` | `#ff6e84` | Danger button variant, "Hot" badge, delete actions |

**Accent (`--primary` #bd93f9) is reserved for exactly these elements:**
1. Primary `Button` variant background
2. Active sidebar navigation item: left-border glow + background tint
3. AI badge chip gradient start color (purple → `--secondary` blue)
4. Section header left-border accent (when content-type is anime)
5. Focus ring outer ring color (`--focus-ring` variable)
6. Active `ToggleGroup` item indicator

**Content-type accent colors — reserved for content-type decoration only:**

| Token | HEX | Contrast vs `--surface-container` | Reserved For |
|-------|-----|------------------------------------|--------------|
| `--accent-anime` | `#bd93f9` | 7.01:1 | Anime card left-border, anime fallback gradient |
| `--accent-manga` | `#f48fb1` | 7.58:1 | Manga card left-border, manga fallback gradient |
| `--accent-game` | `#40e0d0` | 10.31:1 | Game card left-border, game fallback gradient |
| `--accent-news` | `#ffb86c` | 9.93:1 | News card left-border, news fallback gradient |

All four pass WCAG AA 4.5:1 minimum. [Source: design.md §2]

### Glassmorphism Color Tokens (for blurred layers)

| Token | Value | Applied To |
|-------|-------|-----------|
| `--surface-glass` | `rgba(255,255,255,0.03)` | Glassmorphism base layer fill |
| `--surface-elevated` | `#3b3b4a` | DeepDive panel background (non-blurred) |
| `--shadow-lg` | `0 12px 40px rgba(0,0,0,0.4)` | Modal and overlay depth shadow |

### Bold Glass Specification (D-07)

The "bold glass" style applies to: Modal, DeepDive panel, Toast.

| Property | Value |
|----------|-------|
| `backdrop-filter` | `blur(20px)` — default for Modal and Toast |
| `backdrop-filter` | `blur(16px)` — for DeepDive panel (smaller footprint) |
| Border | `1px solid rgba(255,255,255,0.15)` — white/15% luminous border |
| Background | `var(--surface-glass)` + gradient overlay from `--surface-container` |
| Max blur layers active | 2 simultaneously. Each under 15% viewport area. Never on list items. |

**Blur budget priority (D-08 — Claude's Discretion):**
1. Modal (full overlay) — always gets blur(20px); appears alone when open
2. Toast notifications — gets blur(20px); appear above feed, not alongside modal
3. DeepDive panel — gets blur(16px); expanded inline, never simultaneously with modal

GPU risk: Blur on Windows/WebView2 must be validated on mid-tier hardware before Phase 5 completion. [Source: STATE.md §Known Risks]

## Component Inventory

### 7 UI Primitives — CVA Migration (COMP-01)

All 7 primitives move from `const VARIANT_CLASSES = {...}` + `.join(' ')` to `cva()` + `cn()` pattern.

#### Badge

Current variants: `default`, `category`, `hot`, `new`, `count`

New variants to add:
| Variant | Visual Spec |
|---------|-------------|
| `ai` | Purple-to-blue gradient background (`--accent-anime` → `--secondary`), white text weight 600, 11px |
| `content-anime` | `--accent-anime` border + fill at 10% opacity, `--accent-anime` text |
| `content-manga` | `--accent-manga` border + fill at 10% opacity, `--accent-manga` text |
| `content-game` | `--accent-game` border + fill at 10% opacity, `--accent-game` text |
| `content-news` | `--accent-news` border + fill at 10% opacity, `--accent-news` text |

Border radius: `9999px` (pill). [Source: design.md §8]

#### Button

Current variants: `primary`, `secondary`, `ghost`, `danger`

New variants to add:
| Variant | Visual Spec |
|---------|-------------|
| `neon` | `--primary` border (1px), `--primary-glow` background, `--primary` text, `--glow-primary` box-shadow on hover |
| `glass` | Bold glass base (`--surface-glass` bg + white/15% border), `--on-surface` text |

Sizes: `sm`, `md`, `lg` — unchanged values (see existing SIZE_CLASSES).
Border radius: `0.5rem` (8px). [Source: design.md §8]
Active scale: `active:scale-95` — retain existing micro-interaction.

#### Spinner

Sizes: `sm`, `md`, `lg` — retain existing size classes.
New: Spinner color follows `--primary` (neon purple). No new variants needed.

#### Input

Current: `forwardRef`, single style.

Upgrade spec:
- Background: `var(--surface-container)`, border: `1px solid var(--outline-variant)`
- Focus: border becomes `var(--primary)`, box-shadow becomes `var(--focus-ring)`
- Error state: border becomes `var(--error)`, helper text in `--error` at 11px/weight-300
- Border radius: `0.5rem`. [Source: design.md §8]

#### ToggleGroup

Current: generic type, a11y `tablist`, ~55 LOC.
Active item indicator: `--primary-soft` background + `--primary` text + 2px bottom border in `--primary`.
No new variants — upgrade existing active style only.

#### Card

Current: `interactive` mode prop.

New variants:
| Variant | Visual Spec |
|---------|-------------|
| `default` | `--surface-container` bg, `--surface-container-highest` border (1px), `0.875rem` radius |
| `glass` | Bold glass base + `--shadow-md` |
| `featured` | `--surface-container` bg + content-type accent left-border (4px solid) |

Border radius: `0.875rem` (14px). [Source: design.md §8]
Hover state: `--surface-container-high` bg + `--shadow-md` + `translateY(-2px)`.

#### Modal

Current: motion animations + a11y hooks (focusTrap, focusReturn, scrollLock) — already integrated.

Upgrade spec:
- Backdrop: `var(--surface-backdrop)` (`rgba(14,14,19,0.97)`) — no blur on backdrop itself
- Dialog panel: bold glass — `blur(20px)` + white/15% border + `--shadow-lg`
- Border radius: `0.75rem`. [Source: design.md §8]
- Focus trap: already wired via `useFocusTrap`. Verify Tab cycles + Escape closes. [Source: COMP-01 success criteria]

### Feature Components

#### DiscoverCard — Poster Mode (COMP-02)

**Cover art thumbnail — always present:**

| Card State | Thumbnail Position | Thumbnail Size |
|------------|-------------------|----------------|
| `collapsed` | Left side of card, fixed width | 56px wide × 84px tall (2:3 ratio) |
| `summary` | Top of card, full card width | Aspect ratio 2:3, max-height 200px |
| `deepdive` | Top of card, full card width | Aspect ratio 2:3, max-height 200px |

**Fallback (no cover art — D-06):**
- Background: `linear-gradient(135deg, var(--accent-{type}) 0%, var(--surface-container-high) 100%)`
- Overlay icon: lucide-react icon matching content type, centered, 32px, `--on-surface-variant` color
  - Anime: `Tv` icon
  - Manga: `BookOpen` icon
  - Game: `Gamepad2` icon
  - News: `Newspaper` icon
- This icon counts as the 1 decorative icon in the decoration budget.

**Content-type left-border accent:**
- 3px solid left border using `--accent-{type}` on the card container
- Applies in all card states

#### Sidebar Navigation — Active State Overhaul (COMP-04)

**Inactive nav item:**
- Icon: lucide-react icon, 20px, `--on-surface-variant` color
- Label: 12px, weight 400, `--on-surface-variant`
- Background: transparent

**Active nav item:**
- Left border: 3px solid `--primary`
- Background: `--primary-soft` (`rgba(189,147,249,0.15)`)
- Icon color: `--primary`
- Label: weight 600, `--primary`
- Box-shadow: `inset 0 0 16px var(--glow-secondary)` — subtle inward glow

**Hover (inactive) nav item:**
- Background: `--surface-hover` (`rgba(255,255,255,0.04)`)
- Icon color: `--on-surface`

**Nav icons — lucide-react mapping:**

| Wing | Current SVG | lucide-react Replacement |
|------|-------------|--------------------------|
| Discover | Search path | `Search` |
| Library | Bookmark path | `Library` |
| Saved | Bookmark path | `Bookmark` |
| Schedule | Calendar path | `CalendarDays` |
| Profile | User path | `User` |

#### DeepDive Panel — Glassmorphism (COMP-03)

- Background: bold glass — `backdrop-filter: blur(16px)` + `var(--surface-glass)` bg + white/15% border
- Border radius: `0.75rem`
- Shadow: `var(--shadow-md)`
- Question buttons: `--surface-container-high` bg, hover `--surface-active`, `--on-surface-variant` text
- Answer text: 13px, weight 400, `--on-surface`, line-height 1.75 (AI summary line-height)

#### Toast — Glassmorphism (COMP-03)

- Background: bold glass — `backdrop-filter: blur(20px)` + `var(--surface-glass)` bg + white/15% border
- Border radius: `0.875rem`
- Shadow: `var(--shadow-lg)`
- Variants follow semantic color: success uses `--accent-game` (cyan), error uses `--error`, info uses `--secondary`

#### AI Badge Chip (COMP-05)

Claude's Discretion — prescribed here:

- Shape: pill (`border-radius: 9999px`)
- Background: `linear-gradient(90deg, var(--accent-anime) 0%, var(--secondary) 100%)` (purple → blue)
- Text: "AI" label, 10px, weight 600, `#ffffff`
- Padding: `2px 8px` (xs vertical, sm horizontal)
- Position: top-right corner of CardSummary section, overlapping the AI summary label row
- This uses the `ai` Badge variant.

#### Section Header Left-Border Accent (COMP-06)

Claude's Discretion — prescribed here:

- Left border: `4px solid var(--primary)` (primary purple for generic headers)
- Content-type headers use `4px solid var(--accent-{type})`
- Padding-left: `12px` (md spacing) from the border
- Title: 18px, weight 600, `--on-surface`, line-height 1.2
- Optional subtitle: 11px, weight 300, `--on-surface-variant`
- Glow: `box-shadow: -4px 0 12px var(--glow-secondary)` on the border side

#### Empty States — Anime Culture Motifs (COMP-07)

**CSS-only motifs (D-09). No image assets.**

| Empty Context | Motif | CSS Technique |
|---------------|-------|--------------|
| No articles in Feed | Manga speedlines | `conic-gradient` from center, `--outline-variant` lines radiating outward, via `::before` |
| No saved articles | Pixel stars | `box-shadow` cluster of 2px dots in `--primary` at opacity 0.4, via `::after` |
| No search results | Sakura petals | 3-4 rotated `border-radius: 50% 0` shapes in `--tertiary` via `::before`/`::after` |
| No schedule items | Dot grid | `radial-gradient` repeating pattern in `--outline-variant` |

**Decoration budget per empty state component:** 1 animated element + 1 gradient + 1 decorative icon.
- The CSS motif counts as the 1 decorative/gradient element.
- A lucide-react icon (e.g. `Inbox`, `Star`, `Search`) is the 1 decorative icon.
- No ambient animation by default (Phase 6 activates motion).

**Phase 6 CSS class hooks (D-10):**
Every empty state component receives these classes on its root element (styles empty in Phase 5):
- `.retro-decoration`
- `.retro-corner-bracket`
- `.retro-scanline`

## Interaction Contracts

### Focus Management

All modals and panels in this phase must pass:
- Tab key cycles focus within the overlay (no focus escape to background)
- Escape key closes the overlay and returns focus to the trigger element
- Implementation: use existing `useFocusTrap` + `useFocusReturn` hooks (already in Modal.tsx)
- [Source: COMP-01 success criteria §5, CONTEXT.md §a11y hooks]

### Hover States

Card hover: `translateY(-2px)` + `var(--shadow-md)` + `--surface-container-high` background.
Button hover: brightness increase for `primary`/`danger`; background overlay for `secondary`/`ghost`/`neon`/`glass`.
Nav item hover: `--surface-hover` background overlay.
Note: Motion (translateY) is Phase 5 CSS only — no `motion` library needed yet. Phase 6 replaces with AnimatePresence.

### State Machine — DiscoverCard

No changes to the `collapsed` → `summary` → `deepdive` state machine logic.
Phase 5 only changes visual presentation of each state (poster thumbnail position, glass panels, AI badge).

## Forbidden Patterns (Enforced in This Phase)

| Forbidden | Use Instead |
|-----------|-------------|
| Hardcoded HEX (`#bd93f9`) | CSS variable (`var(--primary)`) or Tailwind: `bg-(--primary)` |
| Tailwind default colors (`purple-400`, `blue-500`) | CSS variable via `bg-(--var)` arbitrary syntax |
| Inline styles (`style={{ }}`) | Tailwind CSS classes only |
| `React.FC<Props>` component type | Function declaration with explicit return type `React.JSX.Element` |
| Default exports | Named exports only |
| `console.log` | pino logger |
| `any` type | `unknown` + type guard |
| Direct `invoke()` in components | `src/lib/tauri-commands.ts` wrapper |
| Inline SVG path strings in JSX | lucide-react named imports |
| Light mode branches (`dark:`) | Dark-only `:root` variables |
| `backdrop-filter` on list items | Blur only on Modals, DeepDive, Toast |
