## Motion Contract

This section is the primary addition of Phase 6. All motion must be guarded by `useMotionConfig()`.

### Animation Budget (from STATE.md — hard limits)

| Budget | Limit |
|--------|-------|
| Simultaneous entrance animations | 1 at a time |
| Ambient / idle animations | Off by default |
| Stagger scope | First 10 visible items only |
| Decoration budget | 1 animated + 1 gradient + 1 decorative icon per component max |

### Wing Transition (MOTN-01)

| Property | Value | Rationale |
|----------|-------|-----------|
| Pattern | `AnimatePresence` wrapping `renderWing()` switch in `AppShell.tsx` | Matches existing Toast/Modal/DiscoverWing pattern |
| Animation style | Fade + slide-up (`opacity: 0→1`, `y: 8→0`) | Matches existing `fadeSlideIn` variant — no new variant needed |
| Duration | 200ms total (spring stiffness 300, damping 25) | ROADMAP success criteria: "200ms fade-slide" |
| Exit | `opacity: 0`, `y: -4`, `duration: 0.15s` | Asymmetric exit is faster than entrance — snappier feel |
| Scroll position | Reset to top on Wing switch | Simpler than restore; React.lazy re-mounts anyway |
| Reduced motion | `fade` variant from `reduced` map — instant opacity crossfade | Via `useMotionConfig()` |
| Key strategy | `key={activeWing}` on the motion wrapper | Forces AnimatePresence to treat each Wing as distinct |

### Stagger Reveal — Card List (MOTN-02)

| Property | Value | Rationale |
|----------|-------|-----------|
| Pattern | `staggerContainer` + `staggerItem` variants on `ArticleList.tsx` | Existing variants in `motion-variants.ts` — extend stagger interval |
| Stagger interval | 150ms (`staggerChildren: 0.15`) | ROADMAP: "~150ms intervals"; current 40ms is too fast |
| Item animation | `opacity: 0→1`, `y: 8→0`, spring stiffness 300 damping 25 | Existing `staggerItem` variant — no change needed |
| Scope | First 10 rendered items only (`index < 10` guard) | ROADMAP + STATE.md budget |
| Below-fold items | Rendered at full opacity immediately (no animation) | Prevents janky scroll-load |
| Infinite scroll batches | No stagger on pagination batches — items appear at full opacity | Avoids competing with the single entrance animation budget |
| Reduced motion | `noStagger` container + `fade` item — instant load | Via `useMotionConfig()` |

### Hover Depth Feedback (MOTN-04)

Applied universally to all interactive cards and buttons. Implemented as Tailwind utility classes + CSS for non-Tailwind components.

| Element | Transform | Shadow | Glow |
|---------|-----------|--------|------|
| DiscoverCard | `translateY(-2px) scale(1.01)` | `--shadow-md` | none (already has border accent) |
| `ui/Card` | `translateY(-2px)` | `--shadow-md` | none |
| Button (default variant) | `translateY(-1px)` | `--shadow-md` | none |
| Button (neon variant) | `translateY(-1px)` | `--shadow-md` | `box-shadow: 0 0 12px var(--glow-primary)` |
| Sidebar nav item (inactive) | none (layout stability) | none | `--glow-secondary` background |
| Interactive icon button | `scale(1.1)` | none | none |

Transition: `transition: transform 0.15s ease, box-shadow 0.2s ease` on all hover targets.
Reduced motion: `transition: none` applied via `@media (prefers-reduced-motion: reduce)` in `globals.css`.

### Micro-interactions (MOTN-05)

| Action | Animation | Keyframe | Duration |
|--------|-----------|----------|----------|
| Bookmark toggle (on) | Scale pop: `1 → 1.3 → 1` + icon fill changes to `--primary` | Reuse `bookmarkPop` keyframe from `animations.css` | 300ms |
| Bookmark toggle (off) | Scale pop: `1 → 1.15 → 1` + icon fill reverts to `--on-surface-variant` | New `bookmarkUnpop` keyframe | 200ms |
| Like / reaction (on) | Scale pop: `1 → 1.4 → 1` | New `likePop` keyframe (same shape as bookmarkPop, higher peak) | 350ms |
| Article mark-as-read | Opacity drift: `1 → 0.5`, no scale | CSS transition, not keyframe | 200ms |

New keyframes to add to `animations.css`:
- `bookmarkUnpop`: `0% scale(1)`, `40% scale(1.15)`, `100% scale(1)` — 200ms
- `likePop`: `0% scale(1)`, `50% scale(1.4)`, `100% scale(1)` — 350ms

All micro-interaction keyframes gated by `@media (prefers-reduced-motion: reduce) { animation: none }` in CSS.

### Retro Decoration Motifs (MOTN-03)

CSS `::before` / `::after` only — zero JavaScript. Applied to designated components.

| Class | Motif | Applied To | Visual Spec |
|-------|-------|-----------|-------------|
| `.retro-corner-bracket` | Corner brackets (top-left + bottom-right) | DiscoverCard (poster mode), SectionHeader | 8px × 8px L-shaped pseudo-elements. `border-color: var(--primary-glow)` at 40% opacity. `border-width: 1.5px`. No fill. `::before` = top-left, `::after` = bottom-right via `position: absolute` corners |
| `.retro-scanline` | Horizontal scanline overlay | DiscoverCard cover art (poster image area only) | `::before` with `background: repeating-linear-gradient(0deg, transparent, transparent 2px, rgba(0,0,0,0.08) 2px, rgba(0,0,0,0.08) 4px)`. `pointer-events: none`. `z-index: 1`. Contained to image wrapper |
| `.retro-decoration` | Dot grid background pattern | EmptyState components, SectionHeader background | `::before` with `background-image: radial-gradient(circle, var(--primary-glow) 1px, transparent 1px)`. `background-size: 16px 16px`. `opacity: 0.3`. `pointer-events: none`. `z-index: 0` |

All pseudo-elements: `position: absolute`, `pointer-events: none`, `z-index` below content.
Parent element must have `position: relative`.
All opacity values are at or below 40% — ambient, not dominant.

### prefers-reduced-motion (MOTN-06)

Two-layer guard is required. Both must be applied together:

| Layer | Mechanism | Covers |
|-------|-----------|--------|
| JavaScript | `useMotionConfig()` → returns `reduced` variants when `useReducedMotion()` is true | All `motion.div` animations (motion/react library) |
| CSS | `@media (prefers-reduced-motion: reduce)` in `globals.css` | CSS keyframes (`bookmarkPop`, `likePop`, `shimmer`), CSS transitions on hover states |

CSS rule to add to `globals.css` (Phase 6 addition):
```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```
Note: `!important` is explicitly permitted for `prefers-reduced-motion` per design.md forbidden patterns section.
